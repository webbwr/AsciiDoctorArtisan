## Executive Summary

Implement real-time syntax error detection for AsciiDoc documents with visual error/warning indicators, hover explanations, and quick fixes. The system will detect syntax errors, semantic issues, style problems, and best practice violations without impacting editor performance.

---

## Requirements

### Functional Requirements

1. **Error Categories**
   - **Syntax Errors** (red underline)
     - Malformed headings (inconsistent `=` counts)
     - Unclosed blocks (`[source]` without closing `----`)
     - Invalid attribute syntax (`:attr` without closing `:`)
     - Malformed cross-references (`<<missing-target`)
     - Invalid table syntax

   - **Semantic Errors** (red underline)
     - Undefined cross-reference targets
     - Missing include files
     - Invalid attribute references (`{undefined-attr}`)
     - Duplicate section IDs

   - **Warnings** (yellow underline)
     - Deprecated syntax
     - Missing alt text for images
     - Unused attributes
     - Long lines (>120 characters)
     - Empty sections

   - **Style Issues** (blue underline)
     - Inconsistent heading styles
     - Mixed list markers
     - Multiple blank lines
     - Trailing whitespace

2. **Visual Indicators**
   - Colored underlines (red/yellow/blue)
   - Gutter icons (error/warning/info)
   - Error count in status bar
   - Hover tooltips with explanations

3. **Quick Fixes**
   - Auto-fix for common issues
   - Lightbulb icon for fixable errors
   - Multiple fix options when applicable

4. **Diagnostics Panel** (optional)
   - List all errors/warnings
   - Click to navigate to error
   - Filter by severity
   - Sort by line number

### Non-Functional Requirements

1. **Performance**
   - Checking delay: 500ms after typing stops (debounced)
   - Check duration: <100ms for 1000-line document
   - No typing lag or UI freezes
   - Incremental checking (only changed regions)

2. **Usability**
   - Non-intrusive (doesn't block editing)
   - Clear error messages (actionable)
   - Easy to disable (F8 toggle, settings option)
   - Accessibility-friendly (screen reader support)

3. **Quality**
   - 100% test coverage for checker engine
   - Type-safe implementation (mypy --strict)
   - Comprehensive error catalog

## Data Models

### Diagnostic

```python
@dataclass
class Diagnostic:
    """Represents a syntax error, warning, or info message."""

    code: str                    # Error code (e.g., "E001", "W003")
    message: str                 # Human-readable message
    severity: DiagnosticSeverity # ERROR, WARNING, INFO
    line: int                    # Line number (0-indexed)
    column: int                  # Column number (0-indexed)
    end_line: int                # End line (for multi-line errors)
    end_column: int              # End column
    source: str = "asciidoc"     # Source of diagnostic
    quick_fixes: List[QuickFix] = field(default_factory=list)
    related_info: List[str] = field(default_factory=list)

class DiagnosticSeverity(Enum):
    """Diagnostic severity levels."""
    ERROR = 1    # Red underline
    WARNING = 2  # Yellow underline
    INFO = 3     # Blue underline
    HINT = 4     # No underline, gutter icon only
```

### QuickFix

```python
@dataclass
class QuickFix:
    """Represents an automated fix for a diagnostic."""

    title: str                   # Display title (e.g., "Add closing :")
    description: str             # Detailed explanation
    edits: List[TextEdit]        # Text edits to apply
    kind: QuickFixKind           # FIX, REFACTOR, INFO

class QuickFixKind(Enum):
    """Types of quick fixes."""
    FIX = "fix"              # Direct error fix
    REFACTOR = "refactor"    # Code improvement
    INFO = "info"            # Informational action

@dataclass
class TextEdit:
    """Single text edit operation."""

    start_line: int
    start_column: int
    end_line: int
    end_column: int
    new_text: str
```

## Performance Optimization

### Strategy

1. **Debouncing**
   - Wait 500ms after typing stops before checking
   - Cancel pending checks on new text changes

2. **Incremental Checking**
   - Only re-check modified regions (line-based)
   - Cache diagnostics for unchanged lines

3. **Asynchronous Checking**
   - Run checks in background thread (QThread)
   - Don't block editor UI

4. **Rule Optimization**
   - Compile regex patterns once
   - Early exit on irrelevant lines
   - Skip expensive checks for short documents (<100 lines)

### Performance Budget

| Operation | Target | Max |
|-----------|--------|-----|
| Syntax rule check | <10ms | 20ms |
| Semantic check | <30ms | 60ms |
| Style check | <20ms | 40ms |
| Total check time (1000 lines) | <50ms | 100ms |
| Debounce delay | 500ms | 1000ms |

## Error Detection Rules

### Heading Validation

```python
def check_headings(lines: List[str]) -> List[Diagnostic]:
    """Check for malformed headings."""
    diagnostics = []
    for i, line in enumerate(lines):
        if line.startswith('='):
            # Count leading =
            leading = len(line) - len(line.lstrip('='))
            # Count trailing =
            trailing = len(line) - len(line.rstrip('='))

            if leading != trailing:
                diagnostics.append(Diagnostic(
                    code="E001",
                    message=f"Malformed heading: {leading} leading, {trailing} trailing",
                    severity=DiagnosticSeverity.ERROR,
                    line=i,
                    column=0,
                    end_line=i,
                    end_column=len(line),
                    quick_fixes=[
                        QuickFix(
                            title=f"Normalize to {leading} `=` signs",
                            description="Remove trailing `=` signs",
                            edits=[TextEdit(i, 0, i, len(line), line.rstrip('=').rstrip())],
                            kind=QuickFixKind.FIX
                        )
                    ]
                ))
    return diagnostics
```

### Block Validation

```python
def check_blocks(lines: List[str]) -> List[Diagnostic]:
    """Check for unclosed blocks."""
    diagnostics = []
    block_stack = []  # Track open blocks

    for i, line in enumerate(lines):
        # Detect block start
        if line.startswith('[') and line.endswith(']'):
            block_type = line[1:-1]
            if block_type in ('source', 'listing', 'example', 'sidebar'):
                block_stack.append((block_type, i))

        # Detect block delimiter
        elif line.startswith('----') or line.startswith('===='):
            if block_stack and not line.startswith('--'):
                # Closing delimiter
                block_stack.pop()

    # Unclosed blocks
    for block_type, start_line in block_stack:
        diagnostics.append(Diagnostic(
            code="E002",
            message=f"Unclosed {block_type} block",
            severity=DiagnosticSeverity.ERROR,
            line=start_line,
            column=0,
            end_line=start_line,
            end_column=len(lines[start_line]),
            quick_fixes=[
                QuickFix(
                    title=f"Add closing delimiter for {block_type} block",
                    description="Insert closing delimiter at end",
                    edits=[TextEdit(len(lines)-1, 0, len(lines)-1, 0, "\n----")],
                    kind=QuickFixKind.FIX
                )
            ]
        ))

    return diagnostics
```

## Risks & Mitigation

### Risk 1: False Positives

**Risk:** Checker flags valid AsciiDoc as errors.

**Mitigation:**
- Conservative rule design (fewer false positives)
- User feedback loop for refinement
- Easy disable per-file (comment: `// asciidoc-disable`)
- Settings to customize strictness

### Risk 2: Performance Impact

**Risk:** Syntax checking slows down editor.

**Mitigation:**
- Debounced checking (500ms delay)
- Async checking in background thread
- Incremental checking (only changed lines)
- Performance budget enforcement

### Risk 3: Rule Complexity

**Risk:** Maintaining complex validation rules.

**Mitigation:**
- Comprehensive unit tests (100% coverage)
- Clear rule documentation
- Regex pattern library
- Gradual rule additions (start simple)
